## 6. MANAGEMENT & GOVERNANCE

### CloudWatch
**What:** Monitoring and observability service
**Real-life:** Like security cameras and meters monitoring your building
**Key Features:**
- **Metrics:** CPU, network, disk usage
- **Logs:** Centralized log management
- **Alarms:** Trigger actions based on metrics
- **Events/EventBridge:** Respond to system events
**Memory Trick:** "CloudWatch = Watch your cloud = Monitoring"

### CloudTrail
**What:** Audit logs for API calls
**Real-life:** Like a security camera recording who entered which room
**Key Points:**
- Records every API call
- Governance, compliance, risk auditing
- **Memory Trick:** "CloudTrail = Trail of breadcrumbs = Audit logs"

### Config
**What:** Track resource configurations and compliance
**Real-life:** Like taking snapshots of your room to see how it changes over time
**Key Points:**
- Configuration history
- Compliance auditing
**Memory Trick:** "Config = Configuration tracking = What changed?"

### Systems Manager
**What:** Operational insights and management
**Real-life:** Like a control panel for all your servers
**Key Features:**
- **Session Manager:** Access EC2 without SSH
- **Parameter Store:** Store configuration data
- **Patch Manager:** Automate patching
**Memory Trick:** "Systems Manager = Manage all systems = Central control"

### Trusted Advisor
**What:** Best practices recommendations
**Real-life:** Like a consultant reviewing your infrastructure
**Five Categories:**
1. Cost Optimization
2. Performance
3. Security
4. Fault Tolerance
5. Service Limits
**Memory Trick:** "Trusted Advisor = Trusted advice = Best practices"

### Control Tower
**What:** Set up and govern multi-account AWS environment
**Real-life:** Like a blueprint and construction manager for building properly
**Key Points:**
- Landing Zone setup
- Guardrails (preventive and detective)
**Memory Trick:** "Control Tower = Air traffic control = Governs accounts"

### Service Catalog
**What:** Create and manage catalogs of approved IT services
**Real-life:** Like an approved vendor list for purchasing
**Memory Trick:** "Service Catalog = Catalog of approved services"

### AWS Health Dashboard
**What:** View AWS service health and account-specific events
**Two Types:**
- **Service Health Dashboard:** Overall AWS service status
- **Personal Health Dashboard:** Events affecting your account
**Memory Trick:** "Health Dashboard = Check AWS health"
