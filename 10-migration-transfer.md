## 10. MIGRATION & TRANSFER

### Snow Family
**What:** Physical devices to migrate data
**Three Types:**
1. **Snowcone:** 8TB, portable (size of a lunchbox)
2. **Snowball Edge:** 80TB-210TB (size of a suitcase)
3. **Snowmobile:** Up to 100PB (an actual truck!)
**Real-life:** Like moving houses - pack everything into boxes/trucks
**Memory Trick:** "Snow Family = Physical hardware = Move massive data"

### DMS (Database Migration Service)
**What:** Migrate databases to AWS
**Key Points:**
- Source database remains operational
- Supports homogeneous and heterogeneous migrations
- **Memory Trick:** "DMS = Database Move Service"

### Application Discovery Service
**What:** Discover on-premises applications for migration planning
**Memory Trick:** "Discovery Service = Discover what to migrate"

### Application Migration Service (MGN)
**What:** Lift-and-shift (rehost) applications to AWS
**Real-life:** Like moving your physical server to the cloud with minimal changes
**Memory Trick:** "MGN = Migration = Move apps to AWS"

### AWS Transfer Family
**What:** Transfer files using SFTP, FTPS, FTP to/from S3 or EFS
**Memory Trick:** "Transfer Family = Transfer files protocols"

### DataSync
**What:** Automate data transfer between on-premises and AWS
**Real-life:** Like an automated sync between your home computer and cloud storage
**Key Points:**
- Faster than internet upload
- Scheduled transfers
**Memory Trick:** "DataSync = Synchronize data automatically"
